# Output Directory

This directory contains all outputs generated by the analysis, organized by type.

## Directory Structure

```
output/
├── tables/      # LaTeX and HTML tables
├── figures/     # Graphs, plots, and maps (PDF, PNG)
└── logs/        # STATA log files (if generated)
```

## Usage Notes

### Tables
- `.tex` files can be directly included in LaTeX documents using `\input{}`
- `.html` files can be opened in any web browser
- Use `outreg2` or `estout` commands to regenerate

### Figures
- PDF files are vector graphics (high quality, scalable)
- PNG files are raster graphics (suitable for presentations)
- Map files use NUTS-2 2013 definitions

### Regeneration

To regenerate all outputs:
```bash
# Delete output files (optional)
rm output/tables/*.tex
rm output/tables/*.html
rm output/figures/*.pdf
rm output/figures/*.png

# Run analysis
do code/main_analysis.do
```

## File Naming Convention

- Problem-specific files: `Q[problem][subpart]_description.[ext]`
- Tables: descriptive names or `table[N].[tex|html]`
- Figures: descriptive names with problem reference
- Maps: `map_[description].[pdf|gph]`

## Notes

- Most outputs are excluded from version control (see `.gitignore`)
- Existing output files may be overwritten when rerunning the analysis
- Graph styling uses the `grstyle` package for consistent appearance
